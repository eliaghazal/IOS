What is the MAR (Memory Address Register)? What does it store?
Full Answer

The Memory Address Register (MAR) specifies the address of the memory location to be read from or written to in the next memory operation.

When the CPU wants to read or write memory, it first places the target address in the MAR. The memory system then uses this address to find the correct location.

04 REGISTER 🔥 EXAM
What is the PC (Program Counter)? What does it store?
Full Answer

The Program Counter (PC) holds the address of the NEXT instruction to be fetched and executed.

After the CPU fetches an instruction from address N, the PC is immediately incremented to N+1 — it always points ahead to the next instruction. The currently executing instruction is in the IR, not the PC.

🔥 Exact exam MCQ: "The address of the next instruction to be executed by the current process is provided by:" → Answer: Program Counter
💡 Memorize
PC = Points Ahead (NEXT instruction)
Think: PC is a finger pointing forward in a book — it always points to the NEXT page, not the one you're reading now.
PC = "Please Continue" (continue from this address next)

10 CYCLE 🔥 EXAM
What are the steps of the Fetch-Decode-Execute (Instruction) Cycle?
Full Answer
  1. FETCH — Copy the instruction at address [PC] into the MAR → MBR → IR
  2. PC+1 — Immediately increment PC so it points to the next instruction
  3. DECODE — CPU reads/interprets the instruction currently in IR (what operation? what operands?)
  4. EXECUTE — Perform the operation. May involve reading/writing memory (MAR, MBR), arithmetic (AC), or I/O
  5. INTERRUPT CHECK — After execute, check: is there a pending interrupt? If yes, handle it; if no, go back to step 1

Key register flow during Fetch: PC → MAR → (memory read) → MBR → IR, then PC = PC + 1

💡 Memorize
F · P · D · E · I Fetch · PC+1 · Decode · Execute · Interrupt-check
"For Pete's Sake Do It!" — and never forget the interrupt check at the end!
11 FLAGS 🔥 EXAM
What is the OF (Overflow Flag)? When is it set to 1?
Full Answer

The Overflow Flag (OF) is set to 1 when the result of an arithmetic operation has a different sign than the operands — indicating that the result overflowed the representable range.

Two cases for OF = 1:

  • Both operands are POSITIVE but result is NEGATIVE (e.g., +6 + +3 = -7 in 4-bit)
  • Both operands are NEGATIVE but result is POSITIVE

OF = 0 when operands have different signs (one positive, one negative) — no overflow possible in that case.

🔥 Rule: OF = 1 when operands have the SAME sign but the result has a DIFFERENT sign.
💡 Memorize
OF = "Same goes in, Different comes out"
Both positive → result negative = OVERFLOW (OF=1)
Both negative → result positive = OVERFLOW (OF=1)
One of each → can NEVER overflow (OF=0 always in this case)
12 FLAGS
What is the SF (Sign Flag)? When is it set to 1?
Full Answer

The Sign Flag (SF) reflects the sign of the result:

  • SF = 1 → result is NEGATIVE (the most significant bit / MSB of the result = 1)
  • SF = 0 → result is POSITIVE or zero (MSB = 0)

In 2's complement: MSB = 1 means negative, MSB = 0 means positive.

Example: result = 1001 → MSB = 1 → SF = 1 (negative)

Example: result = 0011 → MSB = 0 → SF = 0 (positive)

💡 Memorize
SF = Sign of result = MSB of result
SF = 1 means result starts with 1 = it's negative.
SF = Sign → look at the FIRST (leftmost) bit of the result.
13 FLAGS
What is the ZF (Zero Flag)? When is it set to 1?
Full Answer

The Zero Flag (ZF) is set to 1 when the result of an operation equals exactly zero (all bits = 0).

  • ZF = 1 → result = 0000 (all zeros)
  • ZF = 0 → result ≠ 0 (any non-zero value)

Example: result = 0000 → ZF = 1

Example: result = 1001 → ZF = 0

💡 Memorize
ZF = Zero? = all bits are 0
ZF = 1 only when result = 0000. That's it. Dead simple.
14 FLAGS 🔥 EXAM
How do you solve a flag problem (OF, SF, ZF)? Step-by-step method with example.
Step-by-Step Method
  1. Perform the binary addition/subtraction normally
  2. Determine OF: Do both operands have the same sign (both MSB=0 or both MSB=1)?
    • If YES and result has different sign → OF = 1
    • If NO (different signs) → OF = 0
    • If YES and result has same sign → OF = 0
  3. Determine SF: Look at MSB (leftmost bit) of the RESULT.
    • MSB = 1 → SF = 1 (negative)
    • MSB = 0 → SF = 0 (positive)
  4. Determine ZF: Is the result all zeros?
    • 0000 → ZF = 1
    • anything else → ZF = 0
Worked Example (from actual exam): 0110 + 0011
0110 (= +6, MSB=0, positive) + 0011 (= +3, MSB=0, positive) ────── 1001 (result) OF: Both operands positive (MSB=0 for both) Result MSB = 1 (negative) → SIGN CHANGED → OF = 1 SF: Result MSB = 1 → SF = 1 (result is negative) ZF: Result = 1001 ≠ 0000 → ZF = 0 ANSWER: OF=1, SF=1, ZF=0
💡 Memorize — Flag Check Order
OF · SF · ZF OF = check operand signs vs result sign (same → different = overflow)
SF = look at MSB of result (1=negative)
ZF = is result all zeros? (0000 → yes)

16 CACHE 🔥 EXAM
What is Cache Memory? Who manages it? What are its 6 design parameters?
Full Answer

Cache memory is a small, fast buffer between the CPU and main memory. It stores copies of frequently used data/instructions so the CPU doesn't have to wait for slow main RAM every time.

Managed by: HARDWARE only — completely invisible to the OS and to the programmer. The OS cannot see, control, or directly interact with cache.

Transfer units:

  • CPU ↔ Cache: word transfer (one word at a time)
  • Cache ↔ Main Memory: block transfer (a block of words at a time)
6 Cache Design Parameters
  1. Cache size — How much data it can hold
  2. Block size — How much data is transferred from main memory at once (cache line)
  3. Mapping function — How memory blocks map to cache slots (direct, set-associative, fully associative)
  4. Replacement algorithm — Which block to evict when cache is full (LRU = Least Recently Used is most common)
  5. Write policy — Write-through (write to both cache and RAM simultaneously) vs. Write-back (write to cache only, flush to RAM later)
  6. Number of cache levels — L1 (fastest, smallest) → L2 → L3
💡 Memorize — Cache Parameters
S · B · M · R · W · N Size · Block size · Mapping · Replacement (LRU) · Write policy · Number of levels
"Some Big Machines Replace Worn Numbers"
Also remember: Cache = HARDWARE managed. OS has NO access to it.

19 INTERRUPTS
What happens during the Interrupt Cycle? How are multiple interrupts handled?
Interrupt Cycle (added to Fetch-Decode-Execute)

After every instruction executes, the CPU checks: Is there a pending interrupt?

  • If NO: fetch next instruction (normal)
  • If YES:
    1. Suspend execution of current program (save PC and state)
    2. Load the address of the Interrupt Service Routine (ISR) into the PC
    3. Execute the ISR (handle the interrupt)
    4. Restore the saved state (resume the interrupted program)
Multiple Interrupt Handling — Two Methods
Method 1: Disable Interrupts
While handling one interrupt, all other interrupts are disabled (ignored/queued). Handled strictly sequentially. Simple but can miss urgent interrupts.
Method 2: Priority Scheme
Each interrupt has a priority level. A higher-priority interrupt CAN interrupt a lower-priority interrupt handler. Allows nested interrupt handling.
💡 Memorize
Interrupt Cycle = "Save, Jump, Handle, Restore"
1. Save current state → 2. Jump to ISR → 3. Handle interrupt → 4. Restore and continue
Multiple interrupts: either DISABLE (queue) or PRIORITY (nest). Priority = more flexible.
20 EXECUTION TRACE 🔥 EXAM
How do you trace a machine language program (PC, AC, IR values after each step)?
Method

Given: initial PC value, memory contents (address | instruction), and opcode table.

  1. Start at PC. Fetch instruction at that address → put it in IR. Increment PC (PC = PC + 1).
  2. Decode: read the opcode (first 4 bits). Look up the opcode table to find what operation it means.
  3. Execute: Perform the operation. Update AC if needed.
  4. Move to next instruction (repeat with new PC).
Example (from real exam — Q6):
Opcodes: 0001=LOAD AC, 0010=STORE AC, 0101=ADD to AC, 0011=SUBTRACT from AC Memory: 200: 1940 | 201: 5941 | 202: 2941 | 203: 1941 | ... Start: PC=200, AC=0, IR=? Step 1: Fetch addr 200 → IR=1940, PC=201 LOAD AC from addr 940 → AC = value at mem[940] Step 2: Fetch addr 201 → IR=5941, PC=202 ADD to AC from addr 941 → AC = AC + mem[941] [Continue until program ends → read final PC, AC, IR]
🔥 After final instruction: PC = last_address + 1. IR = last instruction fetched. AC = final computed value. The exam gives you the opcode table — you just execute step by step.
💡 Memorize
Trace method: "Fetch → PC+1 → Decode → Execute → repeat"
At each step write: PC (before fetch) → IR (what you just fetched) → AC (after execute)
Key: PC updates BEFORE execution. IR shows the current instruction. AC accumulates results.
21 MULTIPROGRAMMING
What is multiprogramming (Ch.1 intro)? What problem does it solve?
Full Answer

Multiprogramming is the ability of the OS to keep multiple programs loaded in memory simultaneously and switch the CPU between them.

Problem it solves: In a single-program system, the CPU sits completely idle whenever the program does I/O (e.g., reading a file takes milliseconds, but the CPU could execute millions of instructions in that time). This is enormously wasteful.

Solution: When Program A starts an I/O operation and has to wait, the OS switches the CPU to Program B. While Program B runs, Program A's I/O completes in the background. When A's I/O is done, it goes back to the Ready queue.

Result: CPU stays busy instead of idle → much higher CPU utilization.

💡 Memorize
Multiprogramming = "Never let the CPU idle"
When one program waits for I/O → switch to another. The CPU never sits around doing nothing.
Analogy: A chef with 3 dishes — while one dish bakes in the oven, the chef preps the other two.
22 DATA TRANSFER 🔥 EXAM
What measurements are used for data transfer between CPU↔Cache and Cache↔Main Memory? Why the difference?
Full Answer

CPU ↔ Cache: Data is transferred one word at a time (word transfer). This is fast and fine because cache is extremely fast.

Cache ↔ Main Memory: Data is transferred in blocks (block transfer). A block is multiple words at once. This is done because main memory is much slower than cache — to compensate, the system loads a whole block at once (exploiting the principle of spatial locality: if you need word N, you'll likely need N+1, N+2 soon too).

Why different? The speed gap. Cache is ~60x faster than RAM. Transferring in blocks amortizes the high latency of main memory access.

💡 Memorize
CPU↔Cache = WORD (small, fast) · Cache↔RAM = BLOCK (big, amortize slowness)
Think: Cache is a fast lane — you hand off one item at a time (word). RAM is a slow highway — you load a truck (block) to make the slow trip worthwhile.

26 EVOLUTION
What is Serial Processing? What are its problems?
Full Answer

Serial processing was the earliest form of computing (1940s–50s). Characteristics:

  • No operating system — programmer interacts directly with hardware
  • Programs run one at a time, in sequence
  • Jobs scheduled using sign-up sheets — manual scheduling
  • Console had lights and toggle switches for input/output

Problems:

  • Scheduling problem: If a booked job finishes early, the rest of the time slot is wasted. If it runs over, it's cut off.
  • Setup time: Loading tape, compiling, linking, etc. took a long time — during which the CPU sat idle.
💡 Memorize
Serial Processing = "No OS, one job at a time, manual everything, wasteful"
Imagine a single bathroom in a house — one person at a time, and they have to set it up themselves every time.
27 EVOLUTION
What is Simple Batch Processing? What are its 4 key features?
Full Answer

Simple batch processing introduced a special program called a Monitor (early OS). Jobs were collected in batches and the monitor automatically ran them in sequence, reducing idle time.

4 Key features introduced by batch processing:

  1. Memory protection — Protect the monitor's memory from user programs. User programs cannot overwrite the monitor.
  2. Timer — Prevents a single job from monopolizing the system. If a job runs too long, the timer interrupts and kills it.
  3. Privileged instructions — Certain instructions (like I/O) can only be executed by the monitor (kernel mode), not by user programs.
  4. Interrupts — Allow the OS to give/take control of the processor from user programs.

Jobs used JCL (Job Control Language) to specify resources needed.

💡 Memorize
M · T · P · I Memory protection · Timer · Privileged instructions · Interrupts
"Monitor Throws Programs Into" order — the Monitor introduces all 4.
JCL = Job Control Language (tells batch system what each job needs).
28 EVOLUTION 🔥 EXAM
What is Uniprogramming? What is its main problem?
Full Answer

Uniprogramming (also called Unitasking): only one program runs at a time. When that program does I/O, the CPU sits completely idle waiting for the I/O to finish.

Main problem: CPU utilization is terrible. A typical program might be doing I/O 80–90% of the time. The CPU is idle 80–90% of the time. Enormously wasteful.

🔥 Exam MCQ: "The systems which allow only one process execution at a time are called:" → Unitasking systems (NOT uniprogramming, NOT uniprocessing — the correct exam answer is "Unitasking")

CPU Utilization formula for 1 program: If p = fraction of time doing I/O → CPU util = 1 - p. If p = 0.8 → CPU util = 20%.

💡 Memorize
Uniprogramming = 1 job, CPU idles during I/O = wasteful
The answer in the MCQ is "Unitasking" not "Uniprocessing." Uniprocessing = 1 processor (hardware); Unitasking = 1 task running (software). Know the difference!
29 EVOLUTION 🔥 EXAM
What is Multiprogramming? What is the CPU utilization formula?
Full Answer

Multiprogramming: multiple programs are loaded in memory at the same time. When the running program must wait for I/O, the OS switches the CPU to another program. The CPU is almost never idle.

CPU Utilization Formula (probabilistic model):

p = fraction of time each process spends waiting for I/O n = number of processes (programs) in memory CPU Utilization = 1 - p^n Example: p = 0.8, n = 1 → CPU = 1 - 0.8^1 = 20% Example: p = 0.8, n = 2 → CPU = 1 - 0.8^2 = 1 - 0.64 = 36% Example: p = 0.8, n = 3 → CPU = 1 - 0.8^3 = 1 - 0.512 = 48.8% Example: p = 0.8, n = 5 → CPU = 1 - 0.8^5 = 1 - 0.328 = 67.2%

As n increases (more programs), CPU utilization improves significantly. But benefit decreases with very large n (diminishing returns).

💡 Memorize
CPU util = 1 − p^n
p = I/O fraction · n = number of programs
"1 minus p to the power of n" — more programs = higher utilization = better!
Never forget the ^ (exponent). It's not 1−p×n, it's 1−pn.
30 EVOLUTION
What is Time Sharing?
Full Answer

Time sharing is an extension of multiprogramming where the CPU switches between processes so rapidly that each user gets the impression of having dedicated access to the machine. It enables interactive computing.

  • Multiple users share the computer simultaneously
  • Each gets a time slice (quantum) of CPU time
  • Response time is short enough for interactive use (feels instant)
  • First system: CTSS (Compatible Time-Sharing System) developed at MIT

Key difference from multiprogramming: Multiprogramming maximizes CPU utilization (batch, non-interactive). Time sharing minimizes response time (interactive).

💡 Memorize
Time Sharing = Interactive Multiprogramming = everyone gets a turn quickly
Think of a spinning pizza slicer — every user gets a thin slice, very fast, repeatedly. You feel like you own the whole pie.
First = CTSS at MIT.

33 MEMORY
What is Virtual Memory? What is Paging?
Virtual Memory

Virtual memory allows programs to use more memory than physically available in RAM. The OS gives each process the illusion of a large, private address space. The actual data for pages not currently in RAM is stored on disk (in a swap/page file).

Key: programs use logical (virtual) addresses. The OS maps them to physical addresses in RAM via page tables. If a page isn't in RAM, a page fault occurs and the OS loads it from disk.

Paging

Paging divides both the virtual address space and physical RAM into fixed-size blocks called pages (virtual) and frames (physical).

  • Virtual address = page number + offset
  • OS uses a page table to map virtual page numbers to physical frame numbers
  • Pages not in RAM are stored on disk until needed

Why do we need Suspended states? Explain Blocked/Suspended and Ready/Suspended.
Why Suspension Is Needed

Problem: Imagine ALL processes currently in main memory are in the Blocked state — they are all waiting for I/O. No process is Ready. The CPU is completely idle even though there might be more work to do. The system is wasting time.

Solution: The OS swaps (suspends) one or more Blocked processes out to disk (virtual memory). The freed RAM space can then be used to load a new or previously suspended process that IS ready to run.

Two Suspended States
StateWhereDescription
Blocked/SuspendedDisk (virtual memory)Process was moved to disk while still blocked (still waiting for its event). When event occurs → moves to Ready/Suspended.

What are the 7 steps of a Context Switch? (The S-U-M-S-U-U-R sequence)
  1. SAVE — Save the context of the currently running process (all registers: PC, PSW, general-purpose registers)
  2. UPDATE — Update the PCB of the current process: change its state (to Ready or Blocked), note the reason for switching
  3. MOVE — Move the current process's PCB to the appropriate queue (Ready queue or Blocked queue)
  4. SELECT — Select a new process to run (the scheduler/dispatcher picks from the Ready queue based on scheduling policy)
  5. UPDATE — Update the PCB of the newly selected process: change its state to Running
  6. UPDATE — Update memory management data structures (e.g., page table base register to point to new process's pages)
  7. RESTORE — Restore the context of the new process (load its PC, PSW, and registers from its PCB) → new process begins executing

56 THREAD 🔥 EXAM
What happens to threads when a process is TERMINATED or SUSPENDED?
Process Termination → All threads terminate

When a process is terminated, ALL threads within that process are terminated. Threads share the process's address space. When the process dies, all its resources (memory, files, address space) are deallocated — so all threads lose their execution environment and are also terminated.

A thread's register context and stack are deallocated when the thread TERMINATES — not when it blocks, not when it spawns, not when it unblocks. Only at termination.

61 SMP 🔥 EXAM
What is SMP (Symmetric Multiprocessing)? What are its key characteristics?
Full Answer

SMP (Symmetric Multiprocessing) is a type of parallel computer architecture where:

  • There are two or more processors on a single machine
  • All processors share the same main memory and I/O facilities
  • All processors are connected via a system bus or interconnect
  • Any processor can run the OS kernel (symmetric — no master CPU)
  • The OS performs self-scheduling — any free processor can pick up the next ready process

SMP Design Considerations:

  1. Concurrent processes/threads (multiple running at same time on different CPUs)
  2. Scheduling (must be coordinated across multiple CPUs)
  3. Synchronization (critical sections with multiple CPUs accessing same memory)
  4. Memory management (cache coherence)

62 MICROKERNEL 🔥 EXAM
What is a Microkernel? What are its 3 essential functions? What are its 7 benefits?
Definition

A microkernel is an OS design where the kernel is kept as small as possible — only the absolutely essential functions stay in the privileged kernel space. Everything else (file system, device drivers, virtual memory manager, GUI) runs as separate server processes in user space.

3 Essential Microkernel Functions (ALL kernels must have these)
  1. Low-level memory management — Managing the mapping from virtual to physical addresses (page tables). Does NOT handle full virtual memory policy — just the mechanism.
  2. Inter-Process Communication (IPC) — Mechanism for processes (including user-space servers) to communicate. Since services run as separate processes, they must communicate via IPC.
  3. Basic I/O and interrupt management — Receiving and routing interrupts to the appropriate user-space handler process.
7 Benefits of Microkernel Design
  1. Uniform interfaces — Processes don't need to know if a service is kernel or user-space — they always use the same IPC message interface.
  2. Extensibility — New services can be added in user space without recompiling the kernel.
  3. Flexibility — Existing services can be modified or replaced without affecting the kernel.
  4. Portability — Only the small kernel needs to be rewritten when porting to new hardware.
  5. Reliability — A crash in a user-space driver/service doesn't crash the whole OS (unlike monolithic kernel where a driver bug crashes everything).
  6. Distributed system support — IPC-based design maps naturally to network communication in distributed systems.
  7. Object-oriented design — Modular, clean component design.
🔥 The dispatcher (quantum manager / scheduler) lives in the microkernel. When the number of processes increases, the dispatcher DECREASES the quantum to share CPU fairly.
💡 Memorize — 3 functions
M · I · I Memory (virtual→physical mapping) · IPC · I/O & interrupt mgmt
"Microkernels Include Interrupts" — 3 things. Everything else → user space.
7 benefits: U-EFPRDO → Uniform, Extensible, Flexible, Portable, Reliable, Distributed, OO
63 QUANTUM 🔥 EXAM
What is Quantum (Q)? If the number of processes increases, does the dispatcher increase or decrease Q?
Full Answer

The Quantum (Q) is the maximum number of CPU clock cycles (or instructions) allocated to a process before the dispatcher preempts it and gives the CPU to the next process.

The quantum is managed by the dispatcher (which lives in the microkernel).

If number of processes increases → dispatcher DECREASES the quantum

Why? If Q stays large and there are many processes, each process waits a long time for its turn (poor response time). By decreasing Q, every process gets a turn more frequently — the system remains responsive. If Q is NOT decreased, the system degenerates to essentially FIFO (one process hogs the CPU for a long time).

💡 Memorize
More processes → SMALLER quantum (so everyone gets turns faster)
Think of a pizza: more people = smaller slices (each person still gets a turn, just a smaller piece).
Large Q + many processes = long wait = poor responsiveness = bad.

68 QUANTUM TIMING 🔥 EXAM
How do you calculate execution timing with a quantum Q? Full method with example.
Method
  1. List processes and their instruction counts
  2. Determine bursts: each burst = min(Q, remaining instructions). A full burst = Q instructions. Last burst = remaining instructions (may be <Q).
  3. Write the execution sequence (round-robin): all processes take their burst in order, then repeat until all done.
  4. For timing: multiply each burst by (ns per instruction). Sum up chronologically to find when each process finishes.
Worked Example: A(7), B(5), C(6), D(7), Q=3, 2ns per instruction
Burst breakdown: A: 3, 3, 1 (total=7 ✓) B: 3, 2 (total=5 ✓) C: 3, 3 (total=6 ✓) D: 3, 3, 1 (total=7 ✓) Execution sequence (round-robin): A B C D | A B C D | A D Position: 1:A(3) 2:B(3) 3:C(3) 4:D(3) 5:A(3) 6:B(2) 7:C(3) 8:D(3) 9:A(1) 10:D(1) Time per slot (2ns/instr): Slot 1: A→3×2=6ns | Cumulative: 6 Slot 2: B→3×2=6ns | Cumulative: 12 Slot 3: C→3×2=6ns | Cumulative: 18 Slot 4: D→3×2=6ns | Cumulative: 24 Slot 5: A→3×2=6ns | Cumulative: 30 Slot 6: B→2×2=4ns | Cumulative: 34 ← B FINISHES at 34ns Slot 7: C→3×2=6ns | Cumulative: 40 ← C FINISHES at 40ns Slot 8: D→3×2=6ns | Cumulative: 46 Slot 9: A→1×2=2ns | Cumulative: 48 ← A FINISHES at 48ns Slot 10:D→1×2=2ns | Cumulative: 50 ← D FINISHES at 50ns
💡 Memorize
Step 1: Write bursts. Step 2: Write sequence. Step 3: Multiply each burst × ns/instr. Step 4: Running total.

Partial (final) burst = remaining instructions × ns/instr (NOT Q × ns/instr).

Full Answer

For n processes, the total number of possible execution orderings = n! (n factorial)

  • 2 processes (A, B): 2! = 2 orderings → AB, BA
  • 3 processes (A, B, C): 3! = 6 orderings → ABC, ACB, BAC, BCA, CAB, CBA

Q=2: A(2) → B(2) → A(2) → B(1) Sequence: AA BB AA B (notation: process letter per instruction)

When all processes in main memory are in the Blocked state, the OS can suspend one process by putting it in the Suspend state and transferring it to the hard disk. The freed space in main memory can then be used to bring in another process.

Exam Q13: P1, P2, P3 need resource R. P1 has it. P2 and P3 wait. OS alternately grants to P1 and P3, never P2. What is this called?

This situation is called Starvation.

P2 is ready and able to use the resource, but is perpetually denied access because the OS keeps choosing P1 and P3 alternately. P2 never gets scheduled. This is NOT deadlock (P2 is not waiting for P1 and P3 to release something in a circular dependency) — P2 simply keeps getting skipped.

💡 Memorize
Answer = STARVATION (one word)
P2 CAN run (it's ready) but NEVER gets the chance → starvation. Don't overthink it.

Exam Q14/Q15 (appeared TWICE): When processes interact, two requirements must be satisfied: synchronization and communication. Explain why and give one approach that provides both.

Why synchronization is needed: Processes need to be synchronized to enforce mutual exclusion — ensuring that when one process is in its critical section (accessing shared data), no other process can enter its own critical section simultaneously. Without this, concurrent access corrupts data.

Why communication is needed: Cooperating processes must exchange information — results, signals, or coordination data to work together effectively.

One approach that provides both: Message Passing.

With message passing, a process sends a message (communication) and the receiver must wait for the message (synchronization). Message passing has the further advantage that it lends itself to implementation in distributed systems as well as in shared-memory multiprocessor and uniprocessor systems.

💡 Memorize
Sync = mutual exclusion · Communication = exchange info · Both = Message Passing
This question appeared TWICE. Know the exact phrases: "mutual exclusion," "exchange information," "message passing," "distributed systems."

Throughput — what is it, how do you calculate it, and what does it measure?

Throughput = the number of processes (jobs) completed per unit time. It measures how productive the system is.

Formula: Throughput = Number of jobs ÷ Total time (in hours) Example: 4 jobs done in 30 minutes: 30 minutes = 30/60 = 0.5 hours Throughput = 4 / 0.5 = 8 jobs/hour Example: 4 jobs done in 10 minutes: 10 minutes = 10/60 = 1/6 hours Throughput = 4 / (1/6) = 4 × 6 = 24 jobs/hour

What is the difference between each process having its own Address Space vs threads sharing one?
Process — Each Has Its Own Address Space

Every process has a completely isolated, private address space. Process A cannot read or write Process B's memory (unless they explicitly set up shared memory). This isolation is enforced by the OS via page tables. This means processes are safe from each other but communication is expensive (must go through OS: pipes, sockets, message queues).

Threads — Share Their Process's Address Space

All threads within the same process share the same address space: same code, heap, global variables, and open files. Communication between threads is fast (just read/write shared memory). BUT this sharing requires synchronization (mutexes, semaphores) to prevent race conditions.

🔥 Exam MCQ: "In an OS, each process has its own:" → "Address space and global variables." Note: global variables means WITHIN the process — processes don't share globals with other processes. Threads DO share globals with each other within the same process.
💡 Memorize
Processes = isolated (separate houses) · Threads = shared (roommates in one apartment)
Isolated → safe but slow communication (need to knock on the door). Shared → fast but need to coordinate (don't both use the bathroom at once).

What are the 4 benefits of using Threads?
  1. Economy — Creating and switching between threads is much faster (and cheaper in terms of memory and time) than creating/switching processes. Threads share the process's resources, so no costly duplication.
  2. Resource sharing — Threads share memory and resources of their process automatically. No need for OS-mediated IPC — threads communicate by reading/writing shared memory.
  3. Responsiveness — A multithreaded application can remain responsive even when part of it is blocked. E.g., one thread handles UI, another handles network download.
  4. Utilization of multiprocessor architectures — Threads within a process can run in parallel on different CPUs (with KLT on SMP), giving genuine speedup.

Practical uses of threads: Foreground/background tasks, asynchronous processing, speed on multi-CPU, modular program structure.
